Media Queries

Uma Media Query é uma regra CSS que permite aplicar estilos diferentes dependendo do tamanho da tela (ou de outras características do dispositivo). Com ela, você faz o site se adaptar a celulares, tablets e computadores — isso é o que chamamos de design responsivo.

A sintaxe básica é: @media (condição) { /* estilos */ }. O bloco de estilos dentro dela só é aplicado quando a condição for verdadeira. A condição mais comum é max-width (largura máxima da tela) ou min-width (largura mínima da tela).

Exemplo de código CSS:

/* Estilo padrão — vale para qualquer tela */
.caixa-exemplo {
    background-color: #10439F; /* azul */
    color: white;
    padding: 1rem;
    text-align: center;
}

/* Media Query — só vale quando a tela tiver 600px ou menos */
@media (max-width: 600px) {
    .caixa-exemplo {
        background-color: #C65BCF; /* muda para roxo no celular */
        font-size: 0.9rem;
    }
}

Redimensione a janela do navegador para ver a mudança!

Tela larga = azul  |  Tela estreita (≤ 600px) = roxo

Layouts flexíveis (Flexbox)

O Flexbox (Flexible Box Layout) é uma ferramenta que permite alinhar e distribuir espaço entre itens dentro de um contêiner, mesmo quando seus tamanhos são desconhecidos ou dinâmicos.

Propriedades CSS: flex - flex-basis - flex-direction - flex-grow - flex-flow - flex-shrink - flex-wrap - order.

Resumo das Propriedades CSS:

  • flex — atalho para flex-grow, flex-shrink e flex-basis juntos.
  • flex-basis — define o tamanho inicial do item antes de o espaço ser distribuído.
  • flex-direction — define a direção do eixo principal: row (horizontal) ou column (vertical).
  • flex-grow — define o quanto um item pode crescer em relação aos outros.
  • flex-shrink — define o quanto um item pode encolher quando o espaço é insuficiente.
  • flex-flow — atalho para flex-direction + flex-wrap.
  • flex-wrap — controla se os itens quebram para a linha seguinte quando não cabem.
  • order — altera a ordem visual dos itens sem mudar o HTML.

Propriedades de Alinhamento: justify-content - align-items - align-content - align-self - place-content - place-items - row-gap - column-gap - gap.

Resumo das Propriedades de Alinhamento:

  • justify-content — alinha os itens no eixo principal (ex: centralizar horizontalmente).
  • align-items — alinha os itens no eixo cruzado (ex: centralizar verticalmente).
  • align-content — alinha as linhas do flex quando há múltiplas linhas e espaço sobrando.
  • align-self — sobrescreve o align-items para um item individual.
  • place-content — atalho para align-content + justify-content.
  • place-items — atalho para align-items + justify-items.
  • row-gap — define o espaço entre as linhas.
  • column-gap — define o espaço entre as colunas.
  • gap — atalho para row-gap + column-gap.

                        /* Contêiner */
/* Contêiner — desktop */
.container {
    display: flex;          /* ativa o Flexbox */
    flex-direction: row;    /* itens em linha (padrão) */
    flex-wrap: wrap;        /* quebra linha se não couber */
    justify-content: space-between; /* distribui espaço entre os itens */
    align-items: center;    /* centraliza verticalmente */
    gap: 1rem;              /* espaço entre os itens */
}

/* Itens */
.item {
    flex-grow: 1;           /* pode crescer para preencher o espaço */
    flex-shrink: 1;         /* pode encolher se necessário */
    flex-basis: 200px;      /* tamanho inicial de cada item */
    padding: 1rem;
    text-align: center;
}

/* Responsivo — celular (≤ 600px) */
@media (max-width: 600px) {
    .container {
        flex-direction: column; /* empilha os itens verticalmente */
        align-items: stretch;   /* itens ocupam a largura toda */
    }
}

CSS Grid Layout Profissional

O CSS Grid Layout é um sistema de layout 2D que permite criar estruturas complexas de linhas e colunas para organizar elementos na página. Enquanto o Flexbox é ideal para layouts em uma dimensão (linha ou coluna), o Grid funciona perfeitamente para layouts bidimensionais com mais controle.

Grid vs Flexbox: O Flexbox distribui espaço entre itens em uma dimensão, enquanto o Grid cria uma estrutura de grade com linhas e colunas definidas. Com Grid, você define o layout da página como um todo e posiciona os itens nas células da grade.

Propriedades do Contêiner Grid: display: grid - grid-template-columns - grid-template-rows - grid-template-areas - grid-auto-flow - grid-auto-columns - grid-auto-rows - gap.

Resumo das Propriedades do Contêiner:

  • display: grid — ativa o CSS Grid no contêiner.
  • grid-template-columns — define o tamanho e número de colunas (ex: 1fr 1fr 1fr para 3 colunas iguais).
  • grid-template-rows — define o tamanho e número de linhas.
  • grid-template-areas — define áreas nomeadas para organizar o layout visualmente.
  • grid-auto-flow — controla como os itens são posicionados automaticamente (linha ou coluna).
  • grid-auto-columns — define o tamanho das colunas criadas automaticamente.
  • grid-auto-rows — define o tamanho das linhas criadas automaticamente.
  • gap — define o espaço entre linhas e colunas.

Propriedades dos Itens do Grid: grid-column - grid-row - grid-area - justify-self - align-self.

Resumo das Propriedades dos Itens:

  • grid-column — posiciona um item em colunas específicas (ex: 1 / 3 para ocupar da coluna 1 até a 3).
  • grid-row — posiciona um item em linhas específicas.
  • grid-area — associa um item a uma área nomeada definida no contêiner.
  • justify-self — alinha o item horizontalmente dentro de sua célula.
  • align-self — alinha o item verticalmente dentro de sua célula.

Exemplo de código CSS Grid:

/* Contêiner Grid — desktop */
.grid-container {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr; /* 3 colunas de igual tamanho */
    grid-template-rows: auto auto;      /* 2 linhas automáticas */
    gap: 1rem;                          /* espaço entre célula */
    padding: 1rem;
}

/* Itens do Grid */
.grid-item {
    background-color: #10439F;
    color: white;
    padding: 1rem;
    text-align: center;
}

/* Item especial que ocupa 2 colunas */
.grid-item-destaque {
    grid-column: 1 / 3; /* ocupa da coluna 1 até a 3 */
    background-color: #C65BCF;
}

/* Responsivo — celular (≤ 600px) */
@media (max-width: 600px) {
    .grid-container {
        grid-template-columns: 1fr; /* apenas 1 coluna */
    }

    .grid-item-destaque {
        grid-column: 1; /* ocupa apenas 1 coluna */
    }
}

Usando grid-template-areas para layouts nomeados: 

/* Layout com áreas nomeadas */
.layout {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
    grid-template-areas: 
        "sidebar main sidebar-direita"
        "footer footer footer";
    gap: 1rem;
}

/* Posicionar itens nas áreas */
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.sidebar-direita { grid-area: sidebar-direita; }
.footer { grid-area: footer; }

Unidade fr (fração): A unidade fr distribui o espaço disponível em frações. Por exemplo, 1fr 2fr 1fr cria 3 colunas onde a coluna do meio é 2 vezes maior que as outras.

Manipulação do DOM e Eventos

O DOM (Document Object Model) é uma representação em estrutura de árvore do conteúdo de uma página HTML. Cada elemento HTML se torna um objeto JavaScript que você pode manipular, alterar, ou interagir com eventos como cliques, submissões de formulário, etc.

Quando o navegador carrega uma página HTML, ele cria uma estrutura hierárquica onde cada tag HTML se torna um "nó" (node). O JavaScript pode acessar e modificar esses nós dinamicamente, permitindo criar páginas interativas e responsivas às ações do usuário.

Métodos de Seleção de Elementos: getElementById - querySelector - querySelectorAll - getElementsByClassName - getElementsByTagName.

Resumo dos Métodos de Seleção:

  • getElementById(id) — seleciona um elemento pelo seu atributo ID (o mais rápido).
  • querySelector(seletor) — seleciona o primeiro elemento que corresponde ao seletor CSS.
  • querySelectorAll(seletor) — seleciona todos os elementos que correspondem ao seletor CSS.
  • getElementsByClassName(classe) — seleciona todos os elementos com a classe especificada.
  • getElementsByTagName(tag) — seleciona todos os elementos da tag HTML especificada.

Propriedades para Manipular Conteúdo: textContent - innerHTML - innerText - setAttribute - getAttribute.

Resumo de Manipulação de Elementos:

  • textContent — altera ou obtém apenas o texto do elemento (sem tags HTML).
  • innerHTML — altera ou obtém o conteúdo HTML completo do elemento.
  • innerText — similar ao textContent, mas respeita a visibilidade CSS.
  • setAttribute(nome, valor) — define ou altera um atributo HTML.
  • getAttribute(nome) — obtém o valor de um atributo HTML.

Propriedade classList para Gerenciar Classes CSS: classList.add() - classList.remove() - classList.toggle() - classList.contains().

  • classList.add(classe) — adiciona uma classe CSS ao elemento.
  • classList.remove(classe) — remove uma classe CSS do elemento.
  • classList.toggle(classe) — adiciona ou remove uma classe (alterna).
  • classList.contains(classe) — verifica se o elemento possui uma classe.

Eventos são ações que o usuário realiza, como clicar em um botão, digitar em um campo de texto, ou mover o mouse. Com JavaScript, você pode "escutar" (listen) por esses eventos e executar código em resposta.

Método addEventListener: A forma moderna e recomendada de adicionar eventos a um elemento é usando addEventListener(tipo, função).

Tipos de Eventos Comuns: click - submit - change - input - mouseover - mouseout - keydown - keyup - load.

Exemplo de Seleção e Manipulação do DOM:

/* Selecionar elementos */
const botao = document.getElementById('meu-botao');
const paragrafo = document.querySelector('.texto-principal');
const cards = document.querySelectorAll('.card');

/* Manipular conteúdo */
paragrafo.textContent = 'Novo texto'; /* altera apenas o texto */
botao.innerHTML = '<strong>Clique aqui</strong>'; /* altera com HTML */

/* Manipular atributos */
botao.setAttribute('disabled', 'true');
const classe = botao.getAttribute('class');

/* Manipular classes */
botao.classList.add('ativo');
botao.classList.remove('desativo');
botao.classList.toggle('selecionado');

if (botao.classList.contains('ativo')) {
    console.log('Botão está ativo!');
}

Exemplo de Listener de Eventos:

/* Adicionar listener de clique */
const botao = document.getElementById('enviar');

botao.addEventListener('click', function(evento) {
    console.log('Botão foi clicado!');
    botao.textContent = 'Enviando...';
});

/* Listener de mudança em input */
const campo = document.getElementById('email');

campo.addEventListener('input', function(evento) {
    console.log('Valor digitado:', evento.target.value);
});

/* Listener de submit em formulário */
const formulario = document.getElementById('meu-formulario');

formulario.addEventListener('submit', function(evento) {
    evento.preventDefault(); /* previne o envio padrão */
    console.log('Formulário foi enviado!');
});

/* Listener com arrow function (moderna) */
document.addEventListener('DOMContentLoaded', () => {
    console.log('DOM carregado completamente!');
});

Objeto Event: Quando um evento ocorre, um objeto Event é passado automaticamente para a função callback. Ele contém informações sobre o evento, como event.target (o elemento que disparou o evento), event.type (tipo de evento), e event.preventDefault() (para cancelar o comportamento padrão).

Ciclo de Vida do DOM: O evento DOMContentLoaded é acionado quando o HTML foi completamente carregado e analisado. É o momento certo para adicionar listeners de eventos e inicializar scripts, garantindo que todos os elementos já existem na página.

Validação de Interface de Formulário

A validação de formulário é o processo de verificar se os dados inseridos pelo usuário atendem aos critérios esperados antes de enviar para o servidor. Existem duas abordagens: validação client-side (no navegador com HTML5 e JavaScript) e validação server-side (no servidor).

Validação HTML5: O HTML5 introduziu atributos como required, type, pattern, min, max e maxlength que permitem validação básica sem escrever JavaScript.

Atributos de Validação HTML5:

  • required — torna o campo obrigatório.
  • type="email" — valida automaticamente o formato de e-mail.
  • type="number" — aceita apenas números.
  • type="password" — mascara o texto inserido.
  • pattern="regex" — valida contra uma expressão regular.
  • min e max — definem valores mínimo e máximo.
  • maxlength — limita o número de caracteres.

Validação com JavaScript: Para validações mais complexas, use JavaScript para acessar os valores dos campos e implementar lógica customizada. A propriedade validity fornece informações detalhadas sobre o estado de validação de um elemento.

Exemplo de Formulário com Validação HTML5:

<form id="formulario">
        <label for="nome">Nome:</label>
        <input 
        type="text" 
        id="nome" 
        name="nome" 
        required 
        minlength="3" 
        maxlength="50"
        >

        <label for="email">E-mail:</label>
        <input 
        type="email" 
        id="email" 
        name="email" 
        required
        >

        <label for="idade">Idade:</label>
        <input 
        type="number" 
        id="idade" 
        name="idade" 
        min="18" 
        max="120"
        >

        <label for="telefone">Telefone:</label>
        <input 
        type="tel" 
        id="telefone" 
        name="telefone" 
        pattern="[0-9]{10,11}"
        >

        <button type="submit">Enviar</button>
    </form>

Exemplo de Validação com JavaScript:

const formulario = document.getElementById('formulario');
    const email = document.getElementById('email');

    /* Validação ao submeter o formulário */
    formulario.addEventListener('submit', function(evento) {
        evento.preventDefault();

        /* Validar campo de e-mail */
        if (!email.validity.valid) {
        if (email.validity.valueMissing) {
            alert('E-mail é obrigatório!');
        } else if (email.validity.typeMismatch) {
            alert('Por favor, insira um e-mail válido!');
        }
        return;
        }

        /* Se chegou aqui, tudo está válido */
        alert('Formulário enviado com sucesso!');
        formulario.submit();
    });

    /* Validação em tempo real */
    email.addEventListener('blur', function() {
        if (email.value && !email.validity.valid) {
        email.style.borderColor = 'red';
        } else {
        email.style.borderColor = 'green';
        }
    });

Propriedades do Objeto validity: valid - valueMissing - typeMismatch - patternMismatch - tooShort - tooLong - rangeUnderflow - rangeOverflow.
